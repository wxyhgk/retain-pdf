# 输出结构说明

## 统一输出目录

项目默认把每次任务写入 `output/<job_id>/`。

`job_id` 当前采用统一格式：

```text
YYYYMMDDHHMMSS-xxxxxx
```

例如：

```text
output/
  20260322235425-2b8641/
```

## 典型目录结构

```text
output/
  20260322235425-2b8641/
    originPDF/
    jsonPDF/
    transPDF/
```

## 各目录含义

- `originPDF/`
  原始输入 PDF 或运行时来源文件
- `jsonPDF/`
  OCR 结果、MinerU 解包内容、结构化中间数据
- `transPDF/`
  最终翻译 PDF、翻译中间结果、摘要信息

## API 与前端下载

当前前端和 API 更关注最终交付，因此下载内容主要围绕：

- 最终翻译后的 PDF
- Markdown 与相关资源
- 必要的结构化结果

不再默认把所有中间文件都暴露给前端下载，以减少体积和带宽压力。

## 输出目录的意义

这种结构的好处是：

- 每个任务互相独立
- 方便排查问题
- 方便前后端对接
- 方便二次处理
- 方便后续做任务队列和任务管理

## 适合自动化集成的方式

如果你准备把这个项目接入前端、服务端或者批处理系统，建议统一围绕 `job_id` 和 `output/<job_id>/` 做后续处理。  
这样最稳定，也最容易扩展。
