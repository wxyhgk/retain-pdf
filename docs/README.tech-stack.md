# 技术栈说明

## 总体技术栈

项目采用 Python 作为主语言，以脚本流水线为核心，通过 FastAPI 暴露服务接口，再配合一个轻量前端完成上传、查询和下载。

## 后端

- Python
- FastAPI
- Uvicorn
- Pydantic

后端负责：

- 接收任务
- 调度翻译流程
- 调用 OCR / MinerU
- 组织输出目录
- 提供任务状态查询
- 提供结果下载

## PDF 与排版重建

- Typst
- cmarker
- mitex
- pypdf

这一层主要负责：

- 把翻译结果重新写回页面
- 尽量保留原始版式
- 处理段落、公式、双栏等常见文档结构
- 生成最终 PDF

## OCR 与结构化输入

- 本地 OCR JSON
- MinerU API

这一层主要负责：

- 从原始 PDF 获取结构化内容
- 统一后续翻译与渲染的输入格式

## 翻译模型接入

- DeepSeek API
- OpenAI 兼容接口
- 本地模型服务

项目本身不绑定单一模型。  
只要接口兼容，就可以接入不同的翻译模型服务。

## 前端

- HTML
- CSS
- JavaScript

前端定位是一个轻量可视化入口，主要负责：

- 上传 PDF
- 提交任务
- 查看状态
- 下载结果

## 工程组织方式

当前仓库大致分为以下几层：

- `scripts/`
  核心流水线与命令行入口
- `Fast_API/`
  服务封装
- `front/`
  前端页面
- `output/`
  所有任务输出
- `Data/` 与 `PDFData/`
  测试数据

## 技术路线特点

- 偏工程化，而不是单个实验脚本
- 偏可组合、可替换，而不是强绑定单一路径
- 偏服务部署能力，而不是只在本地手工跑
