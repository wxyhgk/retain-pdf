# 协作与边界

这篇讲后端协同开发怎么不把结构写乱。

## 模块职责

- `routes/*`
  - 只做 HTTP adapter
- `services/jobs/*`
  - 做任务查询、展示、创建、恢复、控制
- `job_runner/*`
  - 做真实执行链
- `models/*`
  - 只放 DTO 和持久化模型

## 依赖方向

默认方向：

- `routes -> services -> job_snapshot_factory / job_launcher / runtime_gateway / db`
- `job_runner -> db / config / runtime state`
- `models` 不反向依赖 `routes` 或 `services`

## AppState 约定

`AppState` 只放在真正需要运行态协同的地方：

- 路由入口
- 生命周期入口
- 运行态执行层

如果只需要：

- `&Db`
- `&AppConfig`
- `&Path`
- `&RwLock<HashSet<String>>`
- `&Arc<Semaphore>`

就不要再传 `AppState`。

## 常见反模式

- 在 route 里拼业务 view
- 在 models 里做文件系统读取
- 在 job_runner 里继续乱传 `AppState`
- 把 file layout 当公开 API 契约

## 稳定边界

- `provider raw`、`normalized`、`published artifact`、`download API` 是四层不同边界
- `failure` 是正式失败真源
- `runtime.stage_history` 是主时间线真源
- `failure_diagnostic` 只保留兼容投影

## 大模型权限约定

大模型可以参与语义层修复，但不能接管结构层、布局层或产物层。

### 允许做的事

- OCR 噪声的语义归一化
  - 例如明显的 LaTeX 误识别修正
  - 例如把 `\\langlen` 这类稳定可规则化的噪声收敛成确定结果
- 翻译草稿与标题候选
- 轻量分类
  - 标题 / 脚注 / 正文的语义判断
  - 是否需要特殊翻译规则的候选判断

### 只允许给建议，不允许直接落地的事

- 是否缩进
- 是否微调行间距
- 是否调整字号
- 是否改变段落布局
- 是否修改遮盖 / 删除策略

### 明确禁止的事

- 改 bbox
- 改分页
- 改原始结构
- 改任务状态
- 改产物路径
- 绕过确定性规则直接重写渲染输入

### 落地原则

- 大模型输出“候选修复”
- 确定性规则层最终生效
- 只要能稳定规则化的内容，优先进入预处理或归一化层，不要依赖模型临场猜测
- 结构层、布局层、产物层必须保持可测、可回放、可复现

## 提交前检查

改到 `rust_api` 时，至少做对应检查：

- `cargo check --quiet`
- 改了 presentation / query，补测试
- 改了 runner / lifecycle，补测试
