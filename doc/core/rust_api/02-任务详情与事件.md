# 任务详情与事件

这篇讲任务执行过程怎么读。

## 主接口

任务详情：

- `GET /api/v1/jobs/{job_id}`

事件流：

- `GET /api/v1/jobs/{job_id}/events`
- `GET /api/v1/ocr/jobs/{job_id}/events`

## 任务详情里该看什么

重点字段：

- `status`
- `stage`
- `stage_detail`
- `runtime`
- `failure`
- `failure_diagnostic`
- `timestamps`
- `actions`
- `artifacts`
- `ocr_job`
- `invocation`
- `normalization_summary`
- `glossary_summary`
- `log_tail`

推荐前端用法：

- 当前阶段说明看 `stage_detail`
- 当前任务真阶段看 `runtime.current_stage`
- 总耗时看 `runtime.total_elapsed_ms`
- 当前阶段耗时看 `runtime.active_stage_elapsed_ms`
- 过程时间线看 `runtime.stage_history`

## 时间线真源

`runtime.stage_history` 是主时间线真源，不要从 `/events` 反推。

每一项通常包含：

- `stage`
- `detail`
- `enter_at`
- `exit_at`
- `duration_ms`
- `terminal_status`

## 事件流怎么读

事件流适合排障，不是主时间线。

事件项常见字段：

- `seq`
- `ts`
- `level`
- `stage`
- `event`
- `event_type`
- `message`
- `payload`

推荐优先看：

- `event_type`
- `stage`
- `message`
- `payload`

## 生命周期

常见 workflow：

- `book`
- `ocr`
- `translate`
- `render`

常见主阶段：

- `queued`
- `ocr_submitting`
- `ocr_processing`
- `normalization`
- `translation_prepare`
- `translating`
- `rendering`
- `saving`
- `finished`
- `failed`

## 失败协议

失败时优先看：

- `failure.summary`
- `failure.category`
- `failure.root_cause`
- `failure.retryable`
- `failure.suggestion`

兼容字段：

- `failure_diagnostic`
- `log_tail`

## 关键结论

- 详情页主接口是 `GET /api/v1/jobs/{job_id}`
- 时间线看 `runtime.stage_history`
- 事件流看 `GET /api/v1/jobs/{job_id}/events`
- 失败真源看 `failure`
