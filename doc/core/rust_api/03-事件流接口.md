# 事件流接口

## 1. 接口地址

主任务事件流：

`GET /api/v1/jobs/{job_id}/events?limit=50&offset=0`

OCR 子任务事件流：

`GET /api/v1/ocr/jobs/{job_id}/events?limit=50&offset=0`

## 2. 这个接口是做什么的

事件流接口主要用于：

- 调试
- 排查问题
- 观察重试链路
- 看后端在某个阶段到底发生了什么

它不是“过程时间线”的主真源。

这个定位要分清：

- 主时间线：`runtime.stage_history`
- 调试明细：`/events`

## 3. 返回结构

实际返回不是顶层 `items`，而是：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [
      {
        "job_id": "20260404150516-75857c",
        "seq": 1,
        "ts": "2026-04-04T15:05:16Z",
        "level": "info",
        "stage": "queued",
        "event": "job_created",
        "message": "任务已创建",
        "payload": {}
      }
    ],
    "limit": 50,
    "offset": 0
  }
}
```

所以前端读取路径应是：

- `data.items`
- `data.limit`
- `data.offset`

## 4. 每条事件的字段

每条事件固定包含：

- `job_id`
- `seq`
- `ts`
- `level`
- `stage`
- `event`
- `message`
- `payload`

其中：

- `seq` 是任务内单调递增序号
- `ts` 是事件时间
- `level` 通常是 `info / warn / error`
- `stage` 是事件发生时所属阶段
- `event` 是机器可读事件名
- `message` 是人类可读说明
- `payload` 是附加细节

## 5. 常见事件类型

当前应重点识别：

- `job_created`
- `status_changed`
- `stage_updated`
- `stage_transition`
- `stage_progress`
- `retry_scheduled`
- `failure_classified`
- `job_terminal`

兼容历史逻辑时，也会看到：

- `job_error`
- `ocr_child_created`
- `ocr_child_finished`

## 6. 默认是否落盘

当前实现里，事件流是默认落盘能力。

会同时写两处：

- 数据库 `events` 表
- `DATA_ROOT/jobs/{job_id}/logs/events.jsonl`

所以只要任务是正常由当前后端执行的，一般都应该能拿到事件流。

## 7. 为什么页面可能显示“事件流为空”

常见原因只有三类：

1. 前端读错了包装层，把 `data.items` 读成了顶层 `items`
2. 请求的不是这个任务的真实 `job_id`
3. 这是极早期历史任务，事件流持久化还没上线

对于最近的新任务，如果接口返回 `200` 且 `data.items` 非空，前端仍显示空，基本就是第 1 类问题。

## 8. 一个真实返回例子

任务 `20260404150516-75857c` 的接口实际返回就是 `200 OK` 且带事件，说明接口本身正常：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [
      {
        "job_id": "20260404150516-75857c",
        "seq": 1,
        "ts": "2026-04-04T15:05:16Z",
        "level": "info",
        "stage": "queued",
        "event": "job_created",
        "message": "任务已创建",
        "payload": {
          "stage": "queued",
          "status": "queued",
          "workflow": "book"
        }
      }
    ],
    "limit": 50,
    "offset": 0
  }
}
```
