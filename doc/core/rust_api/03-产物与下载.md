# 产物与下载

这篇讲任务产物怎么发现、怎么下。

## 产物清单

主接口：

- `GET /api/v1/jobs/{job_id}/artifacts-manifest`
- `GET /api/v1/ocr/jobs/{job_id}/artifacts-manifest`

它是正式的机器发现入口。

每项常见字段：

- `artifact_key`
- `artifact_group`
- `artifact_kind`
- `ready`
- `file_name`
- `content_type`
- `size_bytes`
- `relative_path`
- `checksum`
- `source_stage`
- `updated_at`
- `resource_path`
- `resource_url`

推荐流程：

1. 先查 manifest
2. 找目标 `artifact_key`
3. 确认 `ready=true`
4. 再用 `resource_path` 或 `resource_url`

## 下载入口

常见下载接口：

- `/pdf`
- `/markdown`
- `/normalized-document`
- `/normalization-report`
- `/download`
- `/artifacts/{artifact_key}`

Markdown 相关注意：

- `/markdown` 默认返回 JSON 包装
- `/markdown?raw=true` 返回原始 Markdown
- `markdown_bundle_zip` 是最适合前端直接下载的包

## 四层边界

后端对外看四层：

- `provider raw`
- `normalized`
- `published artifact`
- `download API`

不要把目录结构当公开契约。

## 排错入口

如果按钮不可点，先看：

- `actions.*.enabled`
- `artifacts.*.ready`
- `artifacts-manifest.items[].ready`

如果下载失败，优先看：

- `failure`
- `runtime`
- `log_tail`
- `/events`
